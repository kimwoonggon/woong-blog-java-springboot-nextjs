Spring Boot Migration Final Parity Audit - 2026-05-17

Scope

This audit covers the final local parity pass for replacing the ASP.NET Core backend with the Java 21 Spring Boot backend while preserving the existing Next.js frontend, Docker/nginx topology, API behavior, and browser workflows.

Changed

  • Replaced the backend runtime with Spring Boot 3.5.x, Java 21, Maven, Flyway, PostgreSQL JDBC persistence, Spring Security session auth, OAuth2 login handoff, CSRF validation, security headers, health checks, media handling, diagnostics, and admin/public API endpoints.
  • Preserved frontend API contracts for public content, admin content, auth, site settings, media uploads, work videos, AI fix flows, batch AI flows, and Docker/nginx routing.
  • Added CQRS-style application handlers for AI and media/work-video flows without adding a mediator library.
  • Updated Docker, nginx, compose, runtime scripts, environment examples, and GitHub Actions from ASP.NET Core/.NET to Spring Boot/Maven while keeping the repository git-flow shape.
  • Fixed parity gaps found by exhaustive Docker-dev Playwright runs, including CORS, auth, public detail context, admin mappers, work-video HLS, ordering, thumbnails, admin pages, footer social settings, stale fetches, legacy title saves, pagination, and thumbnail-persistence waits.

Intentionally Not Changed

  • The Next.js app remains the existing App Router frontend.
  • Historical migration/evidence docs remain historical records.
  • Google OAuth2 still requires production secrets and redirect configuration outside the repository.
  • AI provider behavior remains a deterministic compatibility implementation.

Validations Performed

  • Docker Java 21 backend full suite passed with ./mvnw -q clean -Dtestcontainers.enabled=false test.
  • npm run lint, npm run typecheck, npm run build, and npm test -- --run passed.
  • Docker-dev health checks passed through direct backend and nginx loopback URLs.
  • Playwright exhaustive passed against http://127.0.0.1:13000 with 603 passed and 9 skipped.
  • Focused backend, frontend, and browser reruns passed for every parity gap fixed during the final pass.

Risks And Yellow Flags

  • GitHub-hosted CI was pending when this audit artifact was generated because the branch did not yet exist remotely.
  • The local stack used alternate ports because default ports were occupied.
  • Temporary validation containers/networks may remain on the local Docker host.
  • Production OAuth and secret configuration must be supplied outside the repository.

Recommendation

Commit this migration state, push it to dev, verify GitHub CI, promote the same verified commit to main, and verify main CI.
